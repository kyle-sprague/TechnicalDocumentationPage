What is HTML?

HTML stands for Hyper Text Markup Language and it's job is to provide the structure to webpages. It does this by using Elements which act as building blocks for the page. Using HTML, we can tell the browser if a part of the page is a(n):

  • Heading
  • Paragraph
  • Link
  • Image
  • Section

Below, we will tell you how to use the elements and give examples of the elements you will use the most.

Getting Started

Using elements is the most important part of HTML and, luckily, they all follow the same general pattern making it much eaiser for us. All elements use Tags to describe what they are and all Tags have an opening and closing as shown below.

<h1>This is a main Heading </h1>

The <h1> tag is called the opening tag and the </h1> is the closing tag. Everything inbetween those two is the text that is made into the heading (which is what the h1 means). This is the general stucture that all elements will take on in HTML.

Main Elements

There are hundreds of elements in HTML and even the most experienced Developers still need to look some up every now and then but here is a list of the ones that you will be using most:

  • h1-6 -> Headings
  • p -> Paragraphs
  • a -> Links
  • img -> Images

Headings

Headings are the titles and subtitles we use for each section in our page. They are ranked from 1 to 6 in terms of scope with 1 being the largest and 6 being the smallest. This is an example of a heading of the third size:

<h3>This is the third sized Heading </h3>

Paragraphs

Paragraphs are the simplest element to use. They are simply the text you want in paragraph form on the page. Below is an example of how its used.

<p>Paragraph text</p>

Links

Links require the most out of the main elements because you have to specify where the link points to. The user also needs somewhere to click to follow the link which will be the text you put in between the opening and closing tags. This is how we use the link tag:

<a href = "URL for the link">Text the user clicks to follow link<a>

Images

Images can also be tricky to use but as long as you remember one simple idea you shouldn't have any issues. That idea is that the page always needs to be able to find the image. That means the image location either has to be the image location on the web or where it is saved on your local machine. Keeping that in mind will save you a lot of headache with images. Another property of images is that if the image can't be found for whatever reason, there needs to be text describing what should be there. Thats the purpose of the alt part of the tag. Below is an example of the image tag in use.

<img src ="location of the image" alt= "an alternate text from the image">This is the third sized Heading </h3>

Structure

The second job of HTML besides describing the diferent parts of the page is also structuring the page. We do this by breaking the page into mulitple pieces using a tag called the Division tag. A good example of how this tag is used is if we had a page with multiple articles, we would want to give each article its own division so the page knows they are seperate entities. We can also have divisions inside of divisions for added layers of strucutre. Below is how we use the division tag.

<div>All the content inside the division</div>

What's Next?

Once you have good experience with using HTML to structure your page for the browser you might notice something. The page looks kinda boring. Theres no colors and the text just all seems to be hagning around. This is when you move onto the next technology in Web Development, CSS. CSS is how we chose how the page will look to the user. That is the next recommended thing for you to learn.
